Design target:

- Chat/Gate is the primary first-screen action.
- The first viewport should read as a real application workspace: left
  navigation, central chat workbench, and a compact evidence/OS route rail.
- The right rail surfaces receipts, proofs, routes, cost, and provenance.
- OS operations must be visible as MemoryOS retrieval, CapabilityOS routing,
  GenesisOS divergence, Hive execution, and MyWorld control status.
- The UI should feel like a serious provider-grade agent console, not a
  marketing page or raw log viewer.
- The 2026-05-18 v4 board adds a stronger product split: Chat with AIOS,
  Operating Loop, Evidence Desk, and MemoryOS/CapabilityOS/GenesisOS boards.
  Use that board as the current visual target for end-user Control Center work.
- Dark charcoal is acceptable only with clear semantic color use: teal for
  routing, blue for execution, green for accepted/healthy, amber for review,
  red for stops, violet for divergence.
- Small artifacts must have open/copy controls; long paths should never break
  card geometry.
- Mobile should preserve a chat-first reading order and collapse every OS
  board into one column.
