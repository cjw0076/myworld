# ASC-0067 Unified OS Invocation Pipeline

## Why Now

AIOS has separate OS surfaces, but a new task still does not reliably call all
of them in the intended order. MyWorld can dispatch work, Hive can run
provider loops, MemoryOS can build/review context, CapabilityOS can recommend
routes, and GenesisOS can diverge. The missing piece is a single invocation
pipeline that makes these role calls explicit and auditable for every incoming
goal.

This contract creates that pipeline without pretending all child repo agents
can already execute autonomously. V1 is artifact-first: each OS role must
produce a durable artifact, and MyWorld must fail closed when any role is
missing.

## Pipeline V1

Command:

```bash
python scripts/aios_invoke.py --goal "<goal>" --json
```

Output directory:

```text
.aios/invocations/<invocation_id>/
  goal.json
  genesis/branches.json
  memory/context_request.json
  memory/context_pack.md
  capability/route.json
  hive/execution_plan.json
  dispatch/packets.json
  receipt.json
```

V1 may run in `--plan-only` mode by default. It must not execute child repo
source edits. It may call existing local CLIs to create role artifacts.

Known constraint: the current MyWorld dispatch registry does not yet treat
`GenesisOS` as a first-class inbox/outbox repo. ASC-0067 therefore invokes
GenesisOS directly through its local CLI and records a role artifact. Making
GenesisOS dispatchable through `scripts/aios_dispatch.py`,
`scripts/aios_monitor.py`, and child watchers is a follow-on contract, not a
silent expansion of this one.

## Per-OS Responsibility

### GenesisOS.must_produce

- `genesis/branches.json` generated by:

```bash
cd GenesisOS
python -m genesisos.cli diverge --goal "<goal>" --json
```

Required fields:

- `schema_version: genesisos.v1`
- `authority: speculative_only`
- five branch types: `inversion`, `alien_domain`, `constraint_removal`,
  `failure_as_feature`, `anti_user_prompt`
- `contract_seed` per branch

GenesisOS must not select the winning branch. MyWorld records the branches and
may later choose one for contract drafting.

### MemoryOS.must_produce

- `memory/context_request.json`: what context would be needed for the goal.
- `memory/context_pack.md`: current best-effort context pack. If MemoryOS is
  blocked or no accepted memory exists, it must still produce a graceful
  degraded context pack explaining:
  - no accepted context found, or
  - provider/import blocked, or
  - memory review pending.

V1 may use a MyWorld wrapper around existing `memoryos context build` or a
degraded local context request when the MemoryOS CLI cannot satisfy the query.
It must not auto-accept drafts.

### CapabilityOS.must_produce

- `capability/route.json` generated from existing recommendation surfaces:
  - `recommend --task "<goal>"`
  - provider fallback route if provider failure evidence exists

Required fields:

- recommendation-only flag
- ranked capability ids or provider fallback agents
- risk notes
- no tool execution

### Hive.must_produce

- `hive/execution_plan.json`: a bounded execution plan that names:
  - owner repo
  - candidate worker/provider
  - verification gate
  - stop conditions
  - whether execution is allowed now or plan-only

V1 can be a MyWorld-generated Hive plan artifact if no Hive CLI exists for this
exact composition. It must reference Hive as execution owner only when the
contract scope actually assigns Hive work.

### MyWorld.must_produce

- `scripts/aios_invoke.py`
- `tests/test_aios_invoke.py`
- `docs/AIOS_INVOCATION_PIPELINE.md`
- `dispatch/packets.json` summarizing dispatch-ready work packets, not
  necessarily sending them.
- `receipt.json` with per-OS status:
  - `genesis: passed|degraded|failed`
  - `memory: passed|degraded|failed`
  - `capability: passed|degraded|failed`
  - `hive: passed|degraded|failed`

If any OS artifact is missing, the invocation status is not `passed`.

## Verification Gate

```bash
python -m py_compile scripts/aios_invoke.py
python -m unittest tests/test_aios_invoke.py
python scripts/aios_invoke.py --goal "AIOS should route one task through all OS roles" --json
python scripts/aios_invoke.py --goal "AIOS should route one task through all OS roles" --plan-only --write .aios/invocations/asc-0067-smoke --json
python -m unittest discover -s tests -p 'test_aios_*.py'
python scripts/aios_monitor.py assess --json
```

Pass criteria:

- Invocation creates all seven artifacts listed in Pipeline V1.
- `receipt.json` uses schema `aios.invocation_receipt.v1` and includes
  `invocation_id`, `goal_hash`, `created_at`, `plan_only`, `overall_status`,
  `role_statuses`, `artifact_paths`, `stop_conditions_triggered`, and
  `next_action`.
- `goal.json` preserves the exact submitted goal, a stable hash, and the
  selected control-plane root.
- Genesis output contains five branches and no execution authority.
- Memory output exists even when degraded.
- Capability route is recommendation-only.
- Hive plan contains verification gate and stop conditions.
- Receipt reports each OS role separately.
- Re-running the same plan-only invocation with a fixed output path is
  deterministic except for timestamp fields.
- No files under child repo source trees are modified by the smoke command.
- Any missing role artifact sets `overall_status` to `degraded` or `failed`,
  never `passed`.

Required unit-test cases:

- happy path with mocked GenesisOS and CapabilityOS CLI outputs.
- missing GenesisOS CLI returns `missing_genesis_artifact`.
- MemoryOS unavailable still writes degraded `memory/context_pack.md`.
- CapabilityOS route cannot contain `executes_tools=true`.
- Hive plan cannot set `execute_allowed=true` unless an accepted contract id
  is present.
- artifact paths cannot escape `.aios/invocations/<invocation_id>/`.
- child repo source mtime fixture remains unchanged after plan-only smoke.
