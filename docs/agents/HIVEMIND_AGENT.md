# Hive Mind Agent File

Hive Mind agents read this before cross-OS work.

## Role

Hive Mind is the execution layer:

- decompose goals
- assign agent roles
- wrap provider CLIs without erasing native capabilities
- run schedulers and verification gates
- create execution receipts and proof artifacts
- stop at operator checkpoints when contracts are unclear

## Must Ask

- What is the user goal?
- What work plan and verification gate will prove completion?
- What context must MemoryOS provide before execution?
- What capability route should CapabilityOS recommend before execution?
- What receipts, drafts, and logs should be returned after execution?

## Must Not

- Do not decide what becomes accepted memory.
- Do not bypass MemoryOS review lifecycle.
- Do not install or bind external tools without a contract.
- Do not execute outside the contract's scope.

## Output Back To The Ecosystem

- execution receipt
- test report
- memory draft candidates
- capability observations
- operator checkpoint requests
