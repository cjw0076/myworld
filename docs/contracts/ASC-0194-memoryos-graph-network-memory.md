Per the founder directive, GenesisOS was invoked on the goal. Its divergence
returned five mutation axes (inversion, alien_domain, constraint_removal,
failure_as_feature, anti_user_prompt) and cross-domain analogies (biology's

`inversion` axis produced the load-bearing insight:

> **GoEN's STDP failed because it ran continuously on one timescale. AIOS's
> wake/dream cycle IS the missing timescale separation.** STDP edge

> dream consolidates them slow. GoEN's unfinished research is finished by

> lacked is the dream cycle AIOS already has.

consolidation is not noise to suppress — oscillation marks a genuinely
ambiguous relationship that should stay `draft`, not be promoted.

## The two parts

### A. Structural retrieval (safe — directional message passing)

memoryOS holds a 198K-node / 306K-edge graph but retrieves only by
text-embedding similarity — it ignores the edges. Add GoEN's directional
message passing: a query node pulls signal from graph neighbours, so
retrieval uses structure, not just text. Low risk, high value.

### B. STDP edge consolidation as a dream-phase organ

Edges between memories co-retrieved / co-used are strengthened; unused edges
decay — run **only in the dream cycle**, never during wake. This is the
timescale separation. It honors memoryOS's existing plasticity caution
(`memoryOS/docs/OPERATOR_DESIGN_LEDGER.md` GoEN lesson): consolidation
produces a *draft* edge-weight proposal; promotion needs review; edge
identity history is append-only; oscillating edges stay draft.
